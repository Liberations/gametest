lib/assets 底下是素材资源
ic_box1 是宝箱的图片
ic_door 是宝箱门的图片
ic_box_open 是宝箱打开状态
ic_gift ic_gift2 ic_gift3是宝箱打开后礼物的图片 
ic_heat是血条背景图 
ic_left是移动指示器 
ic_mine是玩家的图片
ic_guaiwu是怪物的图片

需要实现的是一个挖宝闯关小游戏
随机一张 5*8的正方形表格地图 
每个格子随机使用ic_box1 ic_box_open
玩家拥有一定的移动次数与血量
初始化生成地图的时候随机产生一些已打开的盒子 每局游戏只能有一个门 怪物数量随着关卡等级提升而增多
游戏开始时 ic_mine代表玩家从表格地图第一行的上方出现 ic_left是一个向左的指示器显示在玩家上下左右 作为提示 上下右的监听请根据ic_left 旋转获得
滑动屏幕操作上下左右 每次移动步数减1 打开人物所在宝箱的东西 开启后显示对应的物品图片 如果开启的是怪物那么血量减去1
开启的是ic_door那么玩家可以选择进入下一关或者继续探索 
当用户进入门那么进入下一个关卡继续生成随机地图
九宫格底部有宝箱获得礼品显示图片 横向列表显示 显示的是剩余步数 当前关卡等级 剩余血量 
当步数不足或者血量不足 提示玩家前往充值或者送礼获取步数与血量 这里用户点击充值血量增加10 点击送礼步数+10
玩家返回需要保存游戏布局数据 下次进入自动恢复
人物初始化的位置底图应该是ic_box_open 去掉上下左右的箭头提示让人物在每个格子中间显示 玩家移动到对应的格子 格子中的宝箱需要马上打开 步数不足或血量不足不允许移动
